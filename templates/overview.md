# 自定义镜像模板

自定义镜像模板（Template）允许你将常用的 Docker 镜像及启动配置保存为可复用的模板，方便在创建 Pod 时快速选择，无需每次手动填写镜像地址和配置参数。

## 什么是模板？

模板是预配置的 Pod 启动方案，包含以下信息：

- **Docker 镜像**：指定容器所使用的镜像（来自 Docker Hub、私有仓库等）
- **启动命令**：容器启动时自动执行的命令
- **环境变量**：预设的环境变量键值对
- **暴露端口**：容器需要对外开放的端口列表
- **存储配置**：推荐的磁盘大小设置

## 主要特性

- **快速复用**：将常用配置保存为模板，创建 Pod 时一键选用
- **公开模板**：公开模板可供所有用户浏览和使用
- **私有模板**：私有模板仅对你的账户可见，适合保存包含私有镜像的配置
- **支持私有镜像仓库**：可配置私有仓库凭证，Pod 启动时自动拉取镜像

## 创建自定义模板

1. 登录 [Submodel.ai 控制台](https://submodel.ai)
2. 在左侧导航栏中选择 **模板（Templates）**
3. 点击 **创建模板**
4. 填写模板信息：

   **基本信息**
   - **模板名称**：为模板设置一个描述性名称
   - **描述**：简要说明该模板的用途和特点

   **容器配置**
   - **Docker 镜像**：填写完整的镜像地址，例如 `nvidia/cuda:12.1.0-runtime-ubuntu22.04`
   - **镜像类型**：公有镜像或私有镜像
   - **启动命令**：设置容器启动时执行的命令（可选）

   **端口配置**
   - 添加需要暴露的 HTTP 或 TCP 端口

   **环境变量**
   - 添加预设的环境变量（不建议在此保存 API Key 等敏感信息）

5. 选择模板可见性（**公开** 或 **私有**）
6. 点击 **保存** 完成创建

## 使用模板创建 Pod

1. 进入 **Pod** 创建页面
2. 在模板选择步骤中，找到你创建的自定义模板
3. 选择模板后，相关配置会自动填入
4. 根据需要调整 GPU 类型、存储大小等参数
5. 完成 Pod 创建

## 管理已有模板

在 **模板** 列表页面，你可以：

- **查看**所有已创建的模板
- **编辑**模板的基本信息和配置
- **删除**不再需要的模板

## 使用私有镜像仓库

如果你的 Docker 镜像存储在私有仓库中：

1. 在模板创建页面，选择 **私有镜像**
2. 填写仓库认证信息（用户名、密码或访问令牌）
3. 保存后，Pod 启动时将自动使用这些凭证拉取镜像

> **安全提示：** 请避免在模板的环境变量中存储 API 密钥、数据库密码等敏感信息。建议在 Pod 运行时通过安全方式传入敏感配置。
