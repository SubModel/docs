# 供应商合作手册

## SubModel 资源 Hosting 流程及充值结算介绍

适用于将自有服务器、GPU 设备或第三方云资源接入 SubModel 平台进行统一管理。

### 一、准备工作

#### 1.1 工作节点：
- 充足的外网带宽（至少 1Gbps）
- 操作系统：Linux Ubuntu 22.04 LTS 纯净系统
- 安装所需工具：Docker、NVIDIA 驱动（GPU 资源需要 CUDA）
- 提供 SSH root 访问权限
- 硬件：支持虚拟化（KVM/GPU 直通等）

#### 1.2 优化资源（推荐）：
- 如果内网中有可用的存储矩阵服务器或分布式存储服务，请提供相应的访问方式（付费网络存储可提供给用户）。
- 建议在内网中安排一台虚拟机或普通工作服务器，配备大容量存储（至少 10G），并通过 10Gbps 以上带宽连接到工作节点。这用于搭建内网镜像代理服务（此举对调度性能影响很大，可避免频繁从互联网拉取镜像）。

### 二、资源接入

#### 2.1 自托管管理
自托管适合拥有硬件资源并希望保留完全控制权的用户。用户登录平台自行接入设备，只需下载对应操作系统的客户端并按照命令行指引操作即可。

#### 2.2 委托 SSH 访问权限交由运营企业管理
这适合希望委托第三方运维团队全面管理机器的用户。用户需生成 SSH 密钥对，将公钥提供给运营企业，并配置服务器 SSH 访问。此操作需签署服务协议合同，以明确运营的范围和权限。

### 三、充值与结算

#### 3.1 充值
目前支持 stripe（信用卡）和加密货币支付接口。未来将支持 PayPal 等其他支付接口。将根据不同渠道调整不同的充值优惠，以满足用户的多样化需求。

*注意：如果用户（仅限 B2B 用户）需要通过银行转账充值，可与我们签订合同并进行对公转账。我们将通过后台为用户账户充值。*

#### 3.2 结算规则
平台的结算规则为实时结算。消费后实时扣费。对于按时长计费的产品，扣费一次。对于按用量计费的产品，最小计费单位为 1 小时。

- **Credit 结算**：一种基于积分或余额计算的结算方式，与美元的价值比为 1:1。
